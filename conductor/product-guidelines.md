# Product Guidelines

## Tone & Voice
- **Professional & Efficient:** Use language that is clear, helpful, and focused on task completion. Avoid unnecessary jargon where possible, but remain precise when explaining technical workflows.

## Visual Style
- **Minimalist & Modern:** Prioritize a clean, uncluttered layout for all generated outputs (websites, dashboards, PDFs). Focus on usability, generous white space, and a refined color palette.

## Accessibility
- **WCAG 2.1 AA Compliance:** All generated web content and documentation must adhere to WCAG 2.1 Level AA standards. This includes proper color contrast, keyboard navigability, and semantic HTML structure.

## Error Handling
- **User-Friendly & Action-Oriented:** When an error occurs, provide a concise explanation of the problem and clear, actionable steps for the user to recover. Avoid cryptic system messages in the primary user interface.
